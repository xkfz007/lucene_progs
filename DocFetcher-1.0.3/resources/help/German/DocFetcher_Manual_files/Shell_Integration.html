Starten Sie DocFechter durch Rechtsklick auf einen Ordner

Wie es funktioniert:

  • Anstelle einer indexfreien Volltextsuche haben wir "temporäre Indexe" implementiert, weil es schwierig war, die erste Lösung mit der bestehenden Programmarchitektur zu integrieren.
  • Temporäre Indexe werden erstellt durch Rechtsklick auf einen Ordner in Ihrem Dateimanager (z. B. Windows Explorer) und Auswahl von "Suche mit DocFetcher".
  • Dadurch startet DocFetcher und der selektierte Ordner wird automatisch für die Indizierung vorbereitet. Alles, was Sie tun müssen, ist "Grünes Licht" für die Indizierung zu geben, indem Sie auf die "Start"-Schaltfläche klicken. Natürlich können Sie vorher noch die Indizierungs-Optionen ändern.
  • Warten Sie, bis die Indizierung beendet ist.
  • Auf diese Weise erstellte Indexe haben eine "Bei Ende löschen"-Markierung und sind im Suchbereich mit einem [X] gekennzeichnet. Diese Markierung zeigt an, das dieser Index nach Programmende entfernt wird.
  • Wenn Sie Ihre Meinung ändern und einen temporären Index permanent machen wolen, rufen Sie durch Rechtsklick auf den Index das Kontextmenu auf und wählen den Eintrag "Löschen bei Programmende umschalten".

Installation unter Windows

Der "Suche mit DocFetcher"-Kontextmenu-Eintrag wird automatischer erzeugt, wenn Sie DocFetcher mit dem Windows-Installationsprogramm installieren. Dieser Eintrag wird bei Deinstallation auch automatisch wieder entfernt. Wenn Sie die portable Version unter Windows nutzen, ist eine manuelle Installation erforderlich:
  • Um den Kontextmenu-Eintrag zu registrieren, starten Sie DocFetcher auf der Befehlszeile mit dem Parameter "--register-contextmenu", z. B.: C:\Programme\DocFetcher\DocFetcher.exe --register-contextmenu
  • Wenn Sie den DocFetcher-Ordner verschieben, müssen Sie den Kontextmenu-Eintrag erneut registrieren. Der vorherige Eintrag wird überschrieben.
  • Zum Entfernen starten Sie DocFetcher mit "--unregister-contextmenu", z. B.: C:\Programme\DocFetcher\DocFetcher.exe --unregister-contextmenu
  • Beachten Sie, das die "(un)register"-Operationen sprachabhängig sind. Das bedeutet, wenn Sie den Kontextmenu-Eintrag registrieren, danach die Sprachumgebung wechseln (z. B. Englisch -> Deutsch), dann wird der unregister-Aufruf keine Auswirkung haben. Sie müssen zur vorherigen Sprachumgebung zurückschalten, um den Kontextmenu-Eintrag zu entfernen. (Alternativ können sie einfach den "Suche mit DocFetcher"-Eintrag in der Windows-Registry entfernen: HKEY_CLASSES_ROOT -> Directory -> shell -> ...)

Installation unter Linux

Wenn Sie DocFetcher mit einer durch Leerzeichen getrennten Ordnerliste starten, wird DocFetcher diese Ordner direkt nach dem Start in die Indizierungs-Warteschlange stellen und sie mit der Markierung "Bei Ende löschen" versehen. DocFetcher kann einige dieser Parameter abweisen, wenn sie keinen Sinn machen (z. B. nicht existierende Ordner). Wenn ein Pfad Leerzeichen enthält, muss er in Anführungszeichen gesetzt werden.

Um DocFetcher mit Ihrem Dateimanager zu integrieren, müssen Sie wissen, wie ein Kontextmenü-Eintrag in Ihrem Dateimanager erstellt wird, der DocFetcher mit den gewählten Ordnern als Startparametern startet.

So geht es in Nautilus:

  • Installieren Sie das Paket 'nautilus-actions' (z. B. sudo apt-get install nautilus-actions).
  • Gehen Sie zu 'Einstellungen > Konfiguration von Nautilus-Aktionen', klicken Sie auf 'Import/Export'.
  • Importieren Sie die in DocFetcher enthaltene Schema-Datei. Ort der Schema-Datei:
    • Linux Debian package: /usr/share/docfetcher
    • Linux Portable: im DocFetcher-Ordner
  • Wenn Sie die portable Version verwenden, müssen Sie die importierte Konfiguration ein Stück weit verändern: Wählen Sie die Konfiguration, klicken Sie auf 'Bearbeiten' und passen Sie den Pfad zum Programm (DocFetcher.sh) und zur Icon-Datei (icons/docfetcher16.png) an. Das müssen Sie jedesmal machen, wenn Sie den DocFetcher-Ordner verschieben.
  • Zum Entfernen des Kontextmenu-Eintrags löschen Sie ihn einfach in der 'Nautilus Aktions-Konfiguration'.
