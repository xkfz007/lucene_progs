Überspringen von Dateien mittels regulärer Ausdrücke

DocFetcher bietet eine Möglichkeit, bestimmte Dateien mittels regulärer Ausdrücke von der Indexierung auszunehmen. Diese regulären Ausdrücke können in dem Feld "Dateien überspringen" im Dialog Indexierungsoptionen eingefügt werden. Das Zeichen '$' wird benutzt, um zwei reguläre Ausdrücke zu trennen.

Dies ist sicher nicht der richtige Ort für eine vertiefte Diskussion über reguläre Ausdrücke (s. dazu Wikipedia), aber einige praktische Beispiele sollten ausreichen:

Überspringe alle MS Excel-Dateien: .*\.xls

Der "." steht für ein unbekanntes Zeichen. "*" bedeutet das vorhergehende Zeichen, kein- oder mehrmals vorkommend, so dass in diesem Fall ".*" für eine Reihe von 0 oder mehreren unbekannten Zeichen steht. Da der "." bereits als spezielles Zeichen verwendet wird und wir ihn selbst als Zeichen treffen wollen, müssen wir ihn maskieren, indem wir ein "\" davor setzen. Die Zeichen A-Z, a-z und 0-9 stehen für sich selbst. Daher trifft ".*\.xls" auf Dateinamen beliebiger Länge zu, die mit einem Punkt und der Folge "xls" enden.

Einige Lizenztextdateien ausschließen: license\.txt $ License\.txt $ LICENSE\.txt

Wie oben erwähnt, kann das Zeichen "$" verwendet werden, um zwei oder mehr reguläre Ausdrücke zu kombinieren. Als Ergebnis wird dieses Muster alle Dateien mit den Namen "license.txt", "License.txt" und "LICENSE.txt" ausschließen. Wieder muss der Punkt maskiert werden, weil er ein Sonderzeichen ist.

Eine Reihe von Ziffern treffen: journal[0-9]*\.doc

Stellen Sie sich vor, Sie möchten eine Anzahl Word-Dokumente, die mit "journal" beginnen und mit einem Zeitstempel enden (z. B. "journal2008.doc"), ausschliessen. Was Sie aber nicht wollen, ist der Ausschluß von Dateien wie "journalism.doc". Ein Ausdruck wie "journal.*\.doc" passt nicht. Die Lösung ist es, den "." durch "[0-9]" zu ersetzen: Während "." auf jedes Zeichen zutrifft, passt "[0-9]" nur auf eine Ziffer. Daher deckt "[0-9]*" eine Reihe von Ziffern ab.

Zusammenspiel zwischen HTML-Paarung und Dateiausschluß

Wenn die HTML-Paarung aktiviert ist und DocFetcher auf ein HTML-Datei-/Verzeichnis-Paar trifft, werden die Ausschlußmuster nur auf die HTML-Datei angewandt, nicht auf die Dateien im zugehörigen HTML-Verzeichnis. Anders gesagt, die Ausschlußmechanismen arbeiten nicht auf Dateien, sondern auf vollständigen Dokumenten.
