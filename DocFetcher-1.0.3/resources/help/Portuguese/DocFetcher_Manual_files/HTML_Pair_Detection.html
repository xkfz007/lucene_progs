Como a detecção de pares funciona

DocFetcher tenta detectar se um determinado documento HTML tem uma pasta associada e, em caso afirmativo, trata-los como um único documento. A principal razão para a inclusão deste recurso para esconder arquivos dentro de pastas HTML, que, muitas vezes, poluem a listagem de resultados. O mecanismo detecção utilizado imita o mecanismo encontrado no Microsoft Windows 2000/ME/XP, que é explicado aqui . Aqui está um breve resumo: Para cada pasta que tem um dos seguintes sufixos:
  • _archivos
  • _arquivos
  • _bestanden
  • _bylos
  • -Dateien
  • _datoteke
  • _dosyalar
  • _elemei
  • _failid
  • _fails
  • _fajlovi
  • _ficheiros
  • _fichiers
  • -filer
  • .files
  • _files
  • _file
  • _fitxers
  • _fitxategiak
  • _pliki
  • _soubory
  • _tiedostot
DocFetcher verifica se há um arquivo HTML com uma correspondente "nome de arquivo" associado. Se assim for, a pasta e o arquivo será tratado como um único documento. Por exemplo, se há uma pasta chamada "foo_files", o DocFetcher verifica se há um arquivo correspondente com o nome "foo.html", "foo.htm", "foo.shtml", "foo.shtm", e assim por diante. Na maioria das vezes, o emparelhamento pode ser ambíguo, mas como você já pode ver, existem algumas raras ocasiões em que este mecanismo não funciona:
  • E se o DocFetcher encontra esses arquivos: "foo.htm", "foo.html" e "foo_files"? Ele irá associar a pasta com um dos dois arquivos HTML, mas qual é o escolhido é indefinido.
  • E os arquivos "foo.htm", "foo_files" e "foo-Dateien"? Aqui, uma das duas pastas será associada com o arquivo HTML, mas a escolha é indefinido também.
    • Ou
  • imagine a seguinte situação: "foo.htm", "foo.html", "foo_files", "foo-Dateien". O arquivo que está ligado à qual pasta? Duas associações serão feitas, mas não há nenhuma garantia de que eles estão corretas.
Então, aqui estão algumas dicas sobre como organizar o seu documento no repositório a fim de evitar tais problemas:
  • Renomeie os seus documentos quando ambiguidade pode acontecer.
  • Não nomeie pastas para que elas se pareçam com pastas HTML. Não dê nomes como "journal_files", "office_files", etc e uma pasta como nome "this_fails" pode realmente levar a aplicação a falhar, tal como indicado pela o nome ( "_fails" é um sufixo de uma pasta HTML, veja acima).
  • Algumas páginas com múltiplos documentos (por exemplo, tutoriais) baixados da web não terão uma estrutura que pode trazer problemas para o DocFetcher. Em vez disso, haverá uma página inicial chamada "index.htm" ou algo parecido, o que aponta para as outras páginas. O mecanismo de emparelhamento descrito acima, obviamente, não funciona aqui (não há associações feitas), e, muitas vezes, é uma idéia melhor para indexar estas páginas ou, pelo menos, indexá-los em um índice separado.
