Ignorando arquivos usando expressões regulares

O DocFetcher provê uma forma de excluir determinados arquivos de indexação utilizando expressões regulares. As expressões regulares podem ser inseridas caixa de texto do "Ignorar arquivos" no painel Opções de Indexação . O caracter '$' é usado para separar duas expressões regulares.

Não temos como fazer uma discussão em profundidade sobre expressões regulares (veja mais no Wikipedia por exemplo), mas alguns exemplos práticos pode ser suficiente para que você possa começar a usá-los no DocFetcher:

Excluir todos os arquivos MS Excel: "*\. xls"

O "." representa um caractere desconhecido. O "*" significa "o caractere anterior, zero ou mais vezes ", neste caso," .* " indica uma série de zero ou mais caracteres desconhecidos. Já que o "." já está sendo utilizado como um carácter especial, quando queremos corresponder ao ponto em um nome de arquivo, temos de enganar o programa , colocando um "\" na frente do ponto. As letras A-Z, a-z e 0-9 são interpretadas como letras mesmo. Esta é a razão pela qual ".*\.xls" significa qualquer comprimento de arquivo que termina com um ponto e a seqüência "xls".

Excluir alguns arquivos texto relacionados com licenças: license\.txt $ License\.txt $ LICENSE\.txt

Como mencionado acima, o caracter "$" pode ser usado para combinar duas ou mais expressões regulares. Como resultado, este padrão irá excluir todos os arquivos com o nome "license.txt", "License.txt" e "LICENSE.txt". Novamente, o ponto deve ser precedido por "\" porque é um caractere especial.

Combinando uma seqüência de dígitos: journal[0-9]*\.doc

Suponha que você queira excluir um conjunto de documentos do Word que todos começam com "journal" e termina com uma data ou hora, tipo: journal2008.doc. Mas, o que você não quer é excluir arquivos como "journalism.doc". Uma expressão como "journal.*\.doc" não vai funcionar. A solução é substituir o "." com "[0-9]": Apesar de o "." ser correspondente a qualquer caractere, "[0-9]" corresponde a um dígito. Por conseguinte, "[0-9]*" corresponderá a uma seqüência de dígitos.

Interação entre emparelhamento e exclusão arquivo HTML

Se o emparelhamento de HTML está ativado e o DocFetcher encontra um par arquivo HTML /pasta, os padrões de exclusão serão aplicados somente para o arquivo HTML, e não para os arquivos dentro da pasta associada HTML. Em outras palavras, o mecanismo de exclusão não funciona em arquivos, mas em documentos como um todo.
