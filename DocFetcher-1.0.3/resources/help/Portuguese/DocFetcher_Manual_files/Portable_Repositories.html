Repositório de Documentos Portátil

O que é a versão portátil do DocFetcher?

  • O DocFetcher Portátil pode ser obtido através do site do projeto .
  • Nenhuma instalação: Você só precisa extrair o conteúdo do arquivo baixado em uma pasta.
  • Pode ser executado a partir de ambos Windows e Linux utilizando os respectivos executores (.exe/.bat,. sh)
  • Se você colocar todos os documentos a serem indexados na pasta ou subpastas do DocFetcher (exceto para a pasta "índices"), você pode mover livremente em torno de todo o seu repositório de documentos com o DocFetcher e os seus índices.

Para que serve?

  • Você pode manter todo o seu repositório de documentos com o DocFetcher e os índices, mesmo depois de reinstalar o sistema operacional e mover o repositório para um lugar diferente.
  • Você pode facilmente mover seu repositório de documentos para outro computador.
  • Você pode colocar o seu repositório de documentos em um CD -ROM, uma unidade USB ou outra mídia removível.
  • Você pode colocá-lo em um volume encriptado (recomendamos o: TrueCrypt ).
  • Um único repositório de documentos pode ser compartilhado em uma rede local, mesmo se houver uma combinação de computadores Linux e Windows.
  • Você pode usar um único repositório de documentos em um sistema dual boot com Windows/Linux.

Dicas de Usabilidade

  • Se você colocar DocFetcher em um CD-ROM, você não será capaz de salvar as alterações com as preferências ou os índices, assim lembre-se de configurar corretamente tudo antes de gravá-lo no CD-ROM.
  • Já que o DocFetcher é um programa Java, ele só pode ser executado em computadores que tenham instalado um Java Runtime Environment, versão 1.5.0 ou mais recente. Assim, se você pretende criar um arquivo em CD-ROM de seu repositório, você provavelmente terá que incluir um instalador do Java Runtime Environment (ou dois, se você está usando o Windows e Linux).
  • Malabarismo com múltiplas instâncias do DocFetcher pode ser menos confusa através da definição de programa alternativo janela títulos para algumas dessas instancias. Basta abrir o arquivo "user.properties" com um editor de texto e digitar um título alternativo na linha "AppNme=".

Limitações

  • Você pode executar ao mesmo tempo várias instâncias da mesma instalação do DocFetcher . No entanto, na versão atual do DocFetcher a instância que terminar por último irá substituir todas as alterações feitas por instâncias que foram fechadas antes. Portanto, se você quiser fazer algumas alterações (com as preferências ou índices) certifique-se de que voce está modificando na instância que tem permissão de escrita e que será a última a ser fechada.
  • Como mencionado acima, o repositório de documentos não será totalmente portátil se alguns dos seus documentos estão em algum lugar fora da pasta do DocFetcher. A razão é que o DocFetcher pode usar caminhos de arquivos relativos em vez de absoluto apenas se os documentos estiverem localizados na sua própria pasta.
  • Documentos no interior da pasta "índices" não será indexado. (Na verdade, você não deve colocar nada nesta pasta, porque pode ser removido!)
  • Cuidado com incompatibilidades de nomes de arquivos entre Linux e Windows. Por exemplo, arquivos em Linux podem conter caracteres como ":" ou "|", mas no Windows não podem. Assim, você só pode mover um repositório de documentos do Linux para o Windows ou vice e versa, se o repositório não contém documentos com nomes incompatíveis.
