DocFetcher 1.0.3

Introduction

DocFetcher is an Open Source desktop search application: It allows you to search the contents of documents on your computer. - You can think of it as Google for your local document repository.

(Hint: If you're reading this document inside DocFetcher and prefer to read it in your web browser instead, click on the "Open in external browser" button right above this panel.)

Create an index: Right-click on the "Search Scope" panel on the left and select "Create Index..." Now choose a folder to be indexed. For starters, this should be a small one with not too many files in it, like, say, 50? After selecting the folder, a configuration window pops up. The default configuration should suffice, so just click on the "Run" button and wait until DocFetcher has finished indexing the documents.

Alternatively, you could drag a folder out of your file manager and drop it on the "Search Scope" panel. Another way of creating indexes is rightclicking on a folder in your file manager. This is described below, under the section "Notable Features".

Perform a search: Enter a search term in the text box above the result panel (the table with the column headers) and hit Enter. The search results will be displayed in the result panel and split up into pages if the number of results exceeds a certain limit. You can navigate through the pages using the arrow buttons to the right of the text box you've entered the search term in.

Filter or sort the results: You may enter some values into the "Minimum / Maximum Filesize" box to filter out files with a file size below the minimum and/or above the maximum value. You can also check or uncheck items in the "Filetypes" and "Search Scope" panel to filter out certain file types or files in specific folders. Click on the column headers of the result panel to sort the results by the chosen criterion.

The preview panel: Below (or to the right of) the result panel you can find the preview panel, which provides a text-only preview with search term highlighting, and, for HTML documents, a simple built-in web browser in addition to that. (You might have to install it first.)

(If you are reading this inside DocFetcher, following the instructions in the next paragraph will make this manual disappear. You can either open it in an external browser with the "Open in external browser" button or restore it later with the '?' button.)

Open a document: Double-click on one of the items on the result panel if you think you've found what you've been looking for. By default, DocFetcher will automatically hide in the System Tray after opening a file. DocFetcher will also hide in the System Tray if you click on the "Minimize to system tray" button. To bring it back, either click on the System Tray icon or press the "global hotkey" (Ctrl + F8 by default).

Index updates: If documents in the document repository are added, removed or modified, the corresponding indexes have to be updated. This is done automatically in two ways: (1) If DocFetcher is running, it detects the changes and updates the indexes immediately. (2) If DocFetcher isn't running, the changes are detected by a small daemon program that runs in the background; then the affected indexes are updated the next time DocFetcher starts.
Note: If you are using the portable version of DocFetcher and want the daemon to be run automatically on startup, you have to manually add the daemon executable(s) to your operating system's list of startup programs.

Don't index entire drives: In contrast to other desktop search applications (e.g. Google Desktop), DocFetcher was not designed for indexing entire harddrives (e.g. "C:\"), and may explode if you attempt to do so. DocFetcher expects that you index only what you actually need, which saves both time and diskspace, and also gives much better search results.

Notable Features

Portable document repository: The portable version of DocFetcher allows you to bundle up DocFetcher, your documents and their associated indexes into a single folder, which you can then move around freely, e.g. to removable media, encrypted volumes or other computers. This can also be used for sharing a single document repository across a local area network or across the OS'es of a Windows/Linux dual boot system. Details.

Launch DocFetcher on a folder by rightclicking on it: Sometimes it would be quite convenient to perform a full-text search on a folder just by rightclicking on it in the file manager, without creating any index files. DocFetcher offers a feature (you might call it "temporary indexes") that comes very close to that. Note that this requires manual installation if you're using the Linux and/or portable version of DocFetcher. See here for more.

Detection of HTML pairs: Pairs of HTML files and associated folders (e.g. "foo.htm" and "foo_files") can be treated as a single document. See here for a description of how this works, the rare situations where it might not work, and how to avoid these situations.

Customizable text file extensions: You can specify which files should be indexed as plain text. This feature may be useful for programmers who want to index source code files. HTML file extensions can be customized also.

Indexing multiple folders at once: There is an "Add folder to queue" button on the top right of the indexing configuration window that allows putting several folders in a queue, in which they will be processed one after another.

Excluding certain files from indexing: You can exclude certain files from indexing by entering one or more regular expression based filename patterns. Details.
