Portable Document Repositories

What is the portable version of DocFetcher?

  • Can be obtained from the project website.
  • No installation: You simply extract the contents of the downloaded archive into a folder.
  • Can be run from both Windows and Linux using the respective launchers (.exe/.bat, .sh)
  • If you put all documents to be indexed into the DocFetcher folder or subfolders of that folder (except for the 'indexes' folder), you can freely move around your entire document repository complete with DocFetcher and the indexes.

What it is good for

  • You can keep the whole document repository complete with DocFetcher and indexes even after reinstalling your operating system and moving the repository to a different place.
  • You can easily move your document repository to another computer.
  • You can put your document repository on a CD-ROM (archiving?), a USB drive or other removable media.
  • You can put it into an encrypted volume (recommended: TrueCrypt).
  • A single document repository can be shared across a local area network, even if there's a mix of Linux and Windows computers.
  • You can use a single document repository on a Windows/Linux dual boot system.

Usability tips

  • Just common sense, but still: If you put DocFetcher on a CD-ROM, you won't be able to save changes to the preferences or the indexes, so remember to properly configure everything before burning it to the CD-ROM.
  • Since DocFetcher is a Java program, it can only be run on computers on which a Java Runtime Environment, version 1.6.0 or newer, is installed. Thus, if you intend to create a CD-ROM archive of your repository, you would probably want to include a Java Runtime Environment installer (or two if you're working with both Windows and Linux).
  • Juggling with multiple instances of DocFetcher can be made less confusing by setting alternative program window titles for some of these instances. Just open the "user.properties" file with a text editor and enter an alternative title on the "AppName=" line.

Limitations

  • You can run multiple instances of the same DocFetcher installation at the same time. However, in the current version of DocFetcher the instance that terminates last will override all changes made by instances that terminated earlier. Therefore, if you want to make some changes (to the preferences or indexes) from an instance A, either make sure A is the last one that terminates, or make sure A is the only instance with write permission.
  • As mentioned above, the document repository will NOT be fully portable if some of its documents are somewhere outside the DocFetcher folder. The reason is that DocFetcher can use relative file paths instead of absolute ones only if the documents are all located in its own folder.
  • Documents inside the 'indexes' folder will not be indexed. (In fact, you shouldn't put anything into that folder, because it might be deleted!)
  • Beware of filename incompatibilities between Linux and Windows. For example, on Linux filenames can contain characters such as ":" or "|", but on Windows they can't. As a result, you can only move a document repository from Linux to Windows or in the opposite direction if it doesn't contain documents with incompatible filenames. Oh, and special characters such as German umlauts are an entirely different matter...
